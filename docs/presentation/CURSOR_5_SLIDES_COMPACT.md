# FinGuide — פורמט מצגת ל-Cursor (5 שקופיות בלבד · קומפקט)

**העתיקי את כל הקובץ ל-Cursor + הפרומט בסוף.**  
**אל תוסיף שקופיות. אל תשנה סדר. עברית RTL.**

---

## שקופית 1 — POC: הדגשים והמשוב

**כותרת:** POC — "רק OCR, בלי בשר"

**בולטים:**
- FinGuide הדגים: העלאת PDF → חילוץ מספרים מתלוש (OCR)
- **משוב המרצה:** האתר **ללא בשר** — לא נותן ערך מעבר לפענוח
- המשתמש קיבל נתונים — בלי **משמעות**, **פעולה**, או **תמונת מצב**
- חסר: מס, ביטוח, פנסיה/השתלמות, ייבוא אוטומטי, ציון, המלצות

**מסר:** צריך **לעבות** את מה שהאתר נותן — ממפענח מסמכים ל**עוזר פיננסי**

**דובר (שורה אחת):** "ב-POC הוכחנו OCR; המשוב הוביל אותנו להוסיף שכבות ערך אמיתיות."

---

## שקופית 2 — מה הוספנו

**כותרת:** מה בנינו אחרי ה-POC

**בולטים:**
| פיצ'ר | מה נותן למשתמש |
|--------|----------------|
| **ייבוא תלושים מ-Gmail** | תלושים נכנסים אוטומטית מהמייל (קריאה בלבד) |
| **ממצאים — פנסיה והשתלמות** | זיהוי: הצהרה בלי הפקדה, פערי אחוזים, רצף שבור |
| **עוזר מס** | בדיקת שנת מס: תלושים חסרים, 106, מעסיקים, חריגות |
| **המלצות ביטוח** | לפי פרופיל + תלושים — מה כנראה חסר ולמה |
| **ציון פיננסי** | 0–100 + פעולות מומלצות (שקופית 5) |

**מסר:** מ-OCR → **איסוף · פרשנות · המלצה · סיכום**

**דובר:** "חמישה בלוקים שהופכים את האתר לעוזר, לא רק לסורק."

---

## שקופית 3 — איך הפיצ'רים עובדים

**כותרת:** איך זה עובד — זרימה אחת

**בולטים:**

**Gmail → מסמכים**  
חיבור OAuth (`gmail.readonly`) → חיפוש מיילים+PDF תלוש → אותו pipeline כמו העלאה ידנית → "יובא מ-Gmail"

**תלוש → ממצאים (פנסיה / השתלמות)**  
OCR + חילוץ → כללים על כל תלוש → הודעה בעברית (למשל: "במרץ אין הפקדת פנסיה")

**תלושים + פרופיל → עוזר מס**  
סריקת שנה: חודשים חסרים, 106, מעסיקים מרובים, מס חריג

**פרופיל + תלושים → ביטוח**  
שאלון אונבורדינג (ילדים, משכנתא, כיסויים קיימים) + כללים → כרטיס המלצה (לא סוכנות)

**הכל יחד → ציון פיננסי**  
חמש קטגוריות (שקופית 5) → ציון + מה לשפר

**דיאגרמה (אופציונלי):**  
`מייל/העלאה → תלושים → ממצאים + מס + ביטוח → ציון`

**דובר:** "אותם מסמכים מזינים את כל השכבות — לא מערכות מנותקות."

---

## שקופית 4 — מה חסר + בדיקות עצמאיות

**כותרת:** מה עוד צריך — ופער הבדיקות העצמאיות

**בולטים — לביצוע מושלם:**
- **דיוק OCR** — כל פורמטי תלוש בישראל
- **הסבר ליד כל ממצא** — "מה זה אומר" + "מה לעשות"
- **Gmail בפרודקשן** — OAuth מלא, יציבות
- **106 / דוח מס** — זרימה שנתית שלמה
- **פרודקשן** — CI/CD, ניטור, אחסון ענן

**בולטים — בדיקות עצמאיות (פער נוכחי):**
- אין עדיין **אימות חיצוני / עצמאי** לתוצאות המערכת
- הממצאים והציון מבוססים על **כללים פנימיים** — לא על ביקורת עצמאית
- **פער:** אין שקיפות "מי בדק את זה" / תהליך audit שמחזק אמון משתמש ורגולציה
- **כיוון עתידי:** בדיקות עצמאיות, תיעוד החלטות, אולי שכבת QA/Compliance או אישור מומחה

**מסר:** המוצר מתקדם — אבל **אמון מלא** דורש גם בדיקות עצמאיות שלא בנינו עדיין

**דובר:** "יש לנו לוגיקה וטסטים — אבל לא עדיין מודל בדיקה עצמאית שמפריד בין 'המערכת אמרה' ל'מאומת'."

## שקופית 5 — הציון הפיננסי: איך נמדד

**כותרת:** ציון פיננסי 0–100 — איך מחושב

**בולטים:**

**סה"כ:** עד **100 נקודות** (ממוצע משוקלל של 5 קטגוריות)  
**מבוסס על:** מסמכים והפרופיל **שהועלו למערכת בלבד** — לא ייעוץ מס/פיננסי

| קטגוריה | נקודות | מה בודק |
|---------|--------|---------|
| **שלמות מסמכים** | 25 | תלושים בשנה, טופס 106, מסמך פנסיה |
| **יציבות שכר** | 20 | עקביות שכר נטו, נתוני מעסיק |
| **מוכנות מס** | 20 | עוזר מס (חוסרים, 106, חריגות) |
| **עקביות פנסיה** | 20 | הפקדות עובד+מעסיק, חודשים חסרים |
| **ביטוחים** | 15 | פרופיל ביטוח + המלצות קריטיות |

**רמות ציון:**
- 85–100 מצוין · 70–84 טוב · 50–69 לשיפור · מתחת 50 דורש טיפול

**בממשק:** ווידג'ט בדשבורד + `/financial-health` + **2 פעולות מומלצות** (למשל: העלה 106)

**דובר:** "ציון אחד שמרכז את כל מה שהוספנו — ככל שמעלים ומשלימים, הציון עולה."

# פרומפט ל-Cursor (העתיקי)

```
צור מצגת בעברית, בדיוק 5 שקופיות, RTL, קומפקט.

מבנה חובה (אל תשנה):
1. POC — משוב "בלי בשר, רק OCR", צורך לעבות את הערך
2. מה הוספנו — Gmail, ממצאים פנסיה/השתלמות, עוזר מס, ביטוח, ציון
3. איך כל פיצ'ר עובד — זרימה קצרה
4. מה חסר + פער בדיקות עצמאיות (audit חיצוני/אמון)
5. ציון פיננסי — 5 קטגוריות, 100 נקודות

כל שקופית: כותרת + 4–6 בולטים מקסימום + משפט דובר אחד.
ללא שקופיות נוספות. ללא טקסט ארוך.
סגנון: מצגת התקדמות אקדמית, ברור, לא טכני מדי.
```
